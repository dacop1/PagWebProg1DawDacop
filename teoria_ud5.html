Unidad 5: Principios de la Programación Orientada a Objetos

Abstracción, encapsulamiento, herencia y polimorfismo a fondo.

Los cuatro pilares de la POO

1. Abstracción

La abstracción consiste en ocultar los detalles complejos y mostrar solo la funcionalidad esencial. En POO, se logra mediante clases abstractas e interfaces. Una clase abstracta puede tener métodos abstractos (sin cuerpo) que las subclases deben implementar.

// Ejemplo de abstracción
abstract class Figura {
    abstract double calcularArea();
}

class Circulo extends Figura {
    private double radio;
    public Circulo(double radio) { this.radio = radio; }
    @Override
    double calcularArea() { return Math.PI * radio * radio; }
}

2. Encapsulamiento

El encapsulamiento protege los datos de un objeto, permitiendo el acceso solo a través de métodos controlados (getters y setters). Se implementa usando modificadores de acceso (private, public, etc.).

public class CuentaBancaria {
    private double saldo; // Solo accesible dentro de la clase

    public double getSaldo() {
        return saldo;
    }

    public void depositar(double monto) {
        if (monto > 0) saldo += monto;
    }

    public void retirar(double monto) {
        if (monto > 0 && monto <= saldo) saldo -= monto;
    }
}

3. Herencia

La herencia permite crear una nueva clase a partir de una existente, reutilizando atributos y métodos. En Java se usa la palabra clave extends. La herencia representa una relación "es-un" (ej. un Perro es un Animal).

class Vehiculo {
    protected String marca;
    public void arrancar() { System.out.println("Vehículo arrancando"); }
}

class Coche extends Vehiculo {
    private int numeroPuertas;
    public void arrancar() { // Sobrescritura
        System.out.println("Coche arrancando");
    }
}

4. Polimorfismo

El polimorfismo permite que un objeto pueda adoptar múltiples formas. En la práctica, significa que una referencia de tipo superclase puede apuntar a un objeto de una subclase, y el método invocado se determina en tiempo de ejecución (dynamic binding).

Vehiculo miVehiculo = new Coche();
miVehiculo.arrancar(); // Llama al método arrancar() de Coche

// Otro ejemplo con interfaces
interface Sonido {
    void hacerSonido();
}

class Perro implements Sonido {
    public void hacerSonido() { System.out.println("Guau"); }
}

class Gato implements Sonido {
    public void hacerSonido() { System.out.println("Miau"); }
}

// Uso polimórfico
Sonido s1 = new Perro();
Sonido s2 = new Gato();
s1.hacerSonido(); // Guau
s2.hacerSonido(); // Miau

Beneficios de la POO

  • Reutilización de código: Herencia y composición.
  • Mantenibilidad: Código más organizado y fácil de modificar.
  • Escalabilidad: Permite añadir nuevas funcionalidades sin afectar lo existente.
  • Modelado cercano a la realidad: Representa entidades del mundo real.

Ejemplo integrador

// Abstracción
abstract class Empleado {
    protected String nombre;
    protected double salarioBase;

    public Empleado(String nombre, double salarioBase) {
        this.nombre = nombre;
        this.salarioBase = salarioBase;
    }

    public abstract double calcularSalario();

    public void mostrarInfo() {
        System.out.println(nombre + " - Salario: " + calcularSalario());
    }
}

// Herencia y polimorfismo
class EmpleadoFijo extends Empleado {
    private double bonificacion;

    public EmpleadoFijo(String nombre, double salarioBase, double bonificacion) {
        super(nombre, salarioBase);
        this.bonificacion = bonificacion;
    }

    @Override
    public double calcularSalario() {
        return salarioBase + bonificacion;
    }
}

class EmpleadoPorHoras extends Empleado {
    private int horasTrabajadas;
    private double tarifaHora;

    public EmpleadoPorHoras(String nombre, double tarifaHora, int horasTrabajadas) {
        super(nombre, 0);
        this.tarifaHora = tarifaHora;
        this.horasTrabajadas = horasTrabajadas;
    }

    @Override
    public double calcularSalario() {
        return tarifaHora * horasTrabajadas;
    }
}

// Uso
Empleado emp1 = new EmpleadoFijo("Carlos", 2000, 500);
Empleado emp2 = new EmpleadoPorHoras("Ana", 15, 160);
emp1.mostrarInfo();
emp2.mostrarInfo();
