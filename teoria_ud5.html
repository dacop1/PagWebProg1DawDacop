Unidad 5: Principios de la POO

Abstracción, encapsulamiento, herencia y polimorfismo a fondo.

Los cuatro pilares de la POO

1. Abstracción

Consiste en ocultar los detalles complejos y mostrar solo la funcionalidad esencial. Se logra mediante clases abstractas e interfaces. El usuario de la clase no necesita saber cómo funciona internamente, solo qué hace.

abstract class Figura {
    abstract double calcularArea();  // Contrato: cada figura calcula su área
}

class Circulo extends Figura {
    private double radio;
    public Circulo(double radio) { this.radio = radio; }

    @Override
    double calcularArea() { return Math.PI * radio * radio; }
}

class Rectangulo extends Figura {
    private double base, altura;
    public Rectangulo(double b, double h) { base = b; altura = h; }

    @Override
    double calcularArea() { return base * altura; }
}

2. Encapsulamiento

Protege los datos de un objeto. Los atributos se declaran private y se accede a ellos solo a través de métodos públicos controlados (getters/setters). Esto previene que datos inválidos entren al objeto.

public class CuentaBancaria {
    private double saldo;  // Solo accesible dentro de la clase

    public double getSaldo() { return saldo; }

    public void depositar(double monto) {
        if (monto > 0) saldo += monto;
        else System.out.println("Monto inválido");
    }

    public void retirar(double monto) {
        if (monto > 0 && monto <= saldo) saldo -= monto;
        else System.out.println("Fondos insuficientes o monto inválido");
    }
}

3. Herencia

Permite crear nuevas clases (subclases) a partir de clases existentes (superclases), reutilizando atributos y métodos. Representa una relación "es-un" (un Coche ES UN Vehículo).

class Vehiculo {
    protected String marca;
    protected int velocidadMax;

    public Vehiculo(String marca, int velocidadMax) {
        this.marca        = marca;
        this.velocidadMax = velocidadMax;
    }

    public void arrancar() {
        System.out.println(marca + " arrancando...");
    }
}

class Coche extends Vehiculo {
    private int numeroPuertas;

    public Coche(String marca, int velMax, int puertas) {
        super(marca, velMax);
        this.numeroPuertas = puertas;
    }

    @Override
    public void arrancar() {
        System.out.println("Coche " + marca + " arrancando con " + numeroPuertas + " puertas");
    }
}

4. Polimorfismo

Un objeto puede adoptar múltiples formas. Una referencia de superclase puede apuntar a cualquier subclase, y el método ejecutado se decide en tiempo de ejecución (dynamic binding).

// Polimorfismo con herencia
Vehiculo v = new Coche("Toyota", 180, 4);
v.arrancar(); // Ejecuta el arrancar() de Coche, no de Vehiculo

// Polimorfismo con interfaces
interface Sonido {
    void hacerSonido();
}

class Perro implements Sonido {
    public void hacerSonido() { System.out.println("Guau"); }
}

class Gato implements Sonido {
    public void hacerSonido() { System.out.println("Miau"); }
}

// Mismo tipo (Sonido), distintos comportamientos
Sonido[] animales = { new Perro(), new Gato() };
for (Sonido a : animales) {
    a.hacerSonido(); // Guau, luego Miau
}

Beneficios de la POO

  • Reutilización de código: herencia evita duplicar lógica.
  • Mantenibilidad: cambios en una clase no rompen otras.
  • Escalabilidad: añadir nuevas funcionalidades sin tocar lo existente.
  • Modelado real: representa entidades del mundo real de forma natural.

Ejemplo integrador completo

// Abstracción + Encapsulamiento
abstract class Empleado {
    protected String nombre;
    protected double salarioBase;

    public Empleado(String nombre, double salarioBase) {
        this.nombre      = nombre;
        this.salarioBase = salarioBase;
    }

    public abstract double calcularSalario(); // Cada tipo lo implementa distinto

    public void mostrarInfo() {
        System.out.printf("%s → Salario: %.2f€%n", nombre, calcularSalario());
    }
}

// Herencia + Polimorfismo
class EmpleadoFijo extends Empleado {
    private double bonificacion;

    public EmpleadoFijo(String nombre, double base, double bono) {
        super(nombre, base);
        this.bonificacion = bono;
    }

    @Override
    public double calcularSalario() { return salarioBase + bonificacion; }
}

class EmpleadoPorHoras extends Empleado {
    private int    horas;
    private double tarifa;

    public EmpleadoPorHoras(String nombre, double tarifa, int horas) {
        super(nombre, 0);
        this.tarifa = tarifa;
        this.horas  = horas;
    }

    @Override
    public double calcularSalario() { return tarifa * horas; }
}

// Uso polimórfico
public class Main {
    public static void main(String[] args) {
        Empleado[] plantilla = {
            new EmpleadoFijo("Carlos", 2000, 500),
            new EmpleadoPorHoras("Ana", 15, 160)
        };

        for (Empleado e : plantilla) {
            e.mostrarInfo(); // Cada uno calcula su salario a su manera
        }
    }
}
