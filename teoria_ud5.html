Unidad 5: Principios de la Programación Orientada a Objetos

Fundamentos avanzados: abstracción, encapsulamiento, herencia y polimorfismo.

Los cuatro pilares de la POO

1. Abstracción

Consiste en simplificar la realidad modelando clases con las características esenciales. Se enfoca en qué hace un objeto, no cómo lo hace.

2. Encapsulamiento

Ocultar el estado interno de un objeto y permitir el acceso solo a través de métodos públicos (getters/setters). Protege la integridad de los datos.

3. Herencia

Permite crear nuevas clases basadas en clases existentes, reutilizando código y estableciendo jerarquías.

4. Polimorfismo

Capacidad de un objeto para tomar múltiples formas. En la práctica, significa que una misma interfaz puede ser utilizada por diferentes tipos de objetos.

Ejemplo en Java

// Abstracción
abstract class Animal {
    abstract void hacerSonido();
}

// Herencia y polimorfismo
class Perro extends Animal {
    void hacerSonido() {
        System.out.println("Guau");
    }
}

class Gato extends Animal {
    void hacerSonido() {
        System.out.println("Miau");
    }
}

// Uso
Animal miMascota = new Perro();
miMascota.hacerSonido(); // Guau
